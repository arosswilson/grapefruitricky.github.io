I'm going to take a shot at expalining it.

May 10, 2015

Today I want to discuss take a shot at explaining "active record". I have heard this term a lot and never really understood it, so it will be helpful for me to try to research and explain it to you. When I google "what is active record" (this is how I start all research nowadays) and it returned a definition from Wikipedia that says "In software engineering, the active record pattern is an architectural pattern found in software that stores in-memory object data in relational databases. It was named by Martin Fowler in his 2003 book Patterns of Enterprise Application Architecture." ...What? Let's see if I can break this down a little.
First, let's start by defining a relational database. A relational database is one where a table represents an entity and each row represents a seperate instance of that entity. The columns contain certain values for each instance. For example, you could have a table for fruit and each row would be a kind of fruit. Then each column could have different attributes about the fruits, like size, color, taste, etc.
Cool, hopefully that makes sense. So, back to the original definition, we now know that active record is concept that stores objects in these databases. Since I'm learning about Ruby on Rails, we'll focus on that as the software. Let's break Ruby on Rails down a little bit, because a lot of peple think Ruby on Rails is one thing, but that's not true. Ruby is an object orient programming language. This is probably what you think of when you think of Ruby on Rails. The Rails part is actually a framework that gives the structure for web pages, services provided on that site, and databases. To go back to the definition, Rails is the framework that uses Active Record to story Ruby objects in relational databases. So since the Rails framework adopts the active record concept, that means classes that are created in Ruby, will typically have a corresponding table, and each instance of that class will it a row on that table. This streamlines the information management on a web app and allows you to quickly store and access information.
